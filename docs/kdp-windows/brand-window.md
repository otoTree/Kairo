# Kairo 品牌窗口规范

## 概述

品牌窗口是 Kairo OS 启动后的第一个可见界面，
兼具欢迎引导、系统状态展示和快速入口功能。
它是用户对 Kairo 的第一印象，需要传达品牌调性：安静、智能、可信赖。

## 行为规范

### 启动流程

1. 系统启动后，品牌窗口作为第一个 KDP Surface 创建
2. Logo 和品牌名先渲染（无动画，直接显示）
3. 系统状态面板实时更新（每 2 秒刷新一次）
4. 用户点击快速入口后，对应窗口启动，品牌窗口保持

### Agent 交互

```
Agent 激活时:
  品牌窗口自动让出焦点
  状态面板 Agent 项更新为 "忙碌"（黄色指示灯）

Agent 完成任务时:
  状态面板可显示最近一次任务摘要
  例如: "✓ 已完成: 安装 3 个依赖包"
```

### 窗口尺寸

```
默认尺寸:  480×560px
最小尺寸:  400×480px
位置:      屏幕居中
可调整:    否（固定尺寸）
```

## 品牌设计注意事项

### 独特性

品牌窗口是 Kairo 与其他 OS 最直观的差异化点：

1. **无壁纸/桌面** — Kairo 不是传统桌面 OS，品牌窗口即是"桌面"
2. **Agent 状态前置** — 系统状态面板突出 Agent 状态，强调 Agent-Native 定位
3. **极简入口** — 只有两个快速入口，不是应用网格/启动器
4. **深色基调** — 与终端/开发者文化契合，传达专业感

### 避免的设计模式

- 不使用渐变背景（避免类似 macOS 桌面壁纸的感觉）
- 不使用动态模糊/毛玻璃效果（避免专利风险 + KDP 性能限制）
- 不使用圆形头像/用户信息区（避免类似 Windows/macOS 登录界面）
- 不使用应用网格/Dock 栏（避免类似 iOS/macOS Launchpad）
