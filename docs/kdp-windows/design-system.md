# Kairo 设计系统

## 设计哲学

Kairo 的视觉语言追求「静谧科技感」——在极简与功能之间取得平衡。
灵感来源于现代玻璃态设计（Glassmorphism）与瑞士排版传统，
但形成独立的品牌识别，避免与任何商业产品产生视觉专利冲突。

### 与 Apple 设计的差异化策略

| 维度 | Apple 风格 | Kairo 风格 | 差异化理由 |
|------|-----------|-----------|-----------|
| 窗口控制 | 红黄绿三色圆点（左上） | 单色胶囊按钮组（右上） | 避免 Traffic Light 商业外观 |
| 圆角半径 | 连续曲率（Squircle） | 标准 CSS border-radius | 避免 Squircle 专利争议 |
| 模糊效果 | 高斯模糊 + 饱和度提升 | 单层半透明 + 纯色叠加 | KDP 原生渲染性能友好 |
| 字体 | SF Pro（专有） | Inter / Noto Sans SC（开源） | 开源许可，无法律风险 |
| 图标 | SF Symbols（专有） | Phosphor Icons（MIT） | 开源图标库 |
| 动效 | 弹簧物理动画 | 线性 ease-out 过渡 | 简化实现，避免专利 |
| 侧边栏 | 半透明材质 | 纯色深底 + 微妙边框 | 降低渲染复杂度 |

## 排版

### 字体栈

```
主字体:    Inter, "Noto Sans SC", system-ui, sans-serif
等宽字体:  "JetBrains Mono", "Noto Sans Mono", monospace
```

> 注意：KDP 原生层当前使用 8x8 位图字体，以下规范适用于 Web 渲染层
> 和未来矢量字体支持后的原生层。

### 字号层级

| 名称 | 大小 | 行高 | 字重 | 用途 |
|------|------|------|------|------|
| Display | 32px | 40px | 600 | 品牌窗口标题 |
| Title | 20px | 28px | 600 | 窗口标题栏 |
| Heading | 16px | 24px | 600 | 区域标题 |
| Body | 14px | 20px | 400 | 正文内容 |
| Caption | 12px | 16px | 400 | 辅助说明 |
| Mono | 13px | 18px | 400 | 终端/代码 |

基于 4px 网格：

```
xs:   4px     组件内部微间距
sm:   8px     紧凑元素间距
md:   12px    默认元素间距
lg:   16px    区域内间距
xl:   24px    区域间间距
2xl:  32px    大区域分隔
3xl:  48px    页面级间距
```

### 窗口内边距

```
窗口标题栏高度:  36px
窗口内边距:      16px（左右），12px（上下）
侧边栏宽度:      220px（文件管理器）
状态栏高度:      28px
```

## 圆角

使用标准 border-radius（非连续曲率），避免 Squircle 专利：

```
none:   0px     表格/列表项
sm:     4px     按钮/输入框/标签
md:     8px     卡片/面板
lg:     12px    窗口/弹窗
xl:     16px    品牌元素/大卡片
```

---

## 阴影

KDP 原生层暂不支持阴影，以下适用于 Web 渲染层：

```
sm:   0 1px 2px rgba(0, 0, 0, 0.3)                          微妙提升
md:   0 4px 12px rgba(0, 0, 0, 0.4)                         卡片/面板
lg:   0 8px 24px rgba(0, 0, 0, 0.5), 0 2px 8px rgba(0,0,0,0.3)  窗口/弹窗
```

## 窗口通用结构

所有 Kairo 窗口共享统一的窗口框架：

```
┌─────────────────────────────────────────────┐
│  [标题文字]                    [─] [□] [×]  │  ← 标题栏 36px
├─────────────────────────────────────────────┤
│                                             │
│              窗口内容区域                      │
│                                             │
│                                             │
├─────────────────────────────────────────────┤
│  状态信息                          辅助操作   │  ← 状态栏 28px（可选）
└─────────────────────────────────────────────┘
```

### 窗口控制按钮

采用右上角胶囊式按钮组，与 Apple 的左上角三色圆点明确区分：

```
按钮样式:  圆角矩形（4px radius），16×16px
按钮间距:  4px
按钮颜色:  默认 Secondary 色，hover 时 Primary 色
排列顺序:  [最小化 ─] [最大化 □] [关闭 ×]
位置:      右上角，距顶部 10px，距右侧 12px
```

关闭按钮 hover 态使用 Error 色（#FF4D6A），其余使用 Primary 文字色。

## 交互状态

```
Default:    正常显示
Hover:      背景色提亮一级（+0.04 亮度）
Active:     背景色压暗（-0.02 亮度）
Focus:      显示 Focus Ring（2px, Kairo Blue 40% 透明度）
Disabled:   整体 40% 透明度，不响应交互
Selected:   背景色 Kairo Blue 15% 透明度
```
