 # v0.6 版本规划（可观测性与诊断）
 
 ## 目标
 - 跨层追踪与结构化日志闭环
 - 诊断快照与问题定位能力完善
 
 ## 交付范围
 - traceId/spanId 贯通 EventBus/IPC/Process
 - 统一日志结构与字段规范
 - 诊断快照一键导出
 
 ## 验收标准
 - 任意进程输出能关联到意图链路
 - 关键事件与错误可回放定位
 - 支持导出最近事件与系统状态

## 纵向切片（E2E）
- 一次失败任务可被追踪与诊断包复现
- 事件、日志与 trace 关联一致

## 关键场景
- 从一次失败任务回溯完整事件链路
- 进程异常退出可定位到触发请求
- 一键导出诊断包用于复现问题

## 非目标
- 全量 APM 或分布式追踪系统
- 性能优化与容量压测
- UI 端复杂可视化
